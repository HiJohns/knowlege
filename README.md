# 我的个人知识库

这是一个由 OpenCode 驱动的个人知识管理系统，用于收集、整理和归档来自互联网的各种有价值的内容。

## 项目背景

在日常阅读中，我们经常会遇到需要保存的内容：
- 微信公众号、知乎等平台的深度文章
- 技术博客、文档教程
- 有价值的社交媒体内容

这些内容通常是 HTML 格式或者带有各种噪音，直接保存既不便于阅读，也不利于后续检索。本知识库的核心思路是：

1. **自动抓取与清洗** - 从 URL 或原始文本中提取纯净内容
2. **脱水处理** - 提取核心观点和关键数据，生成结构化笔记
3. **版本化管理** - 通过 Git 进行版本控制，方便追溯和协作

## 可用命令

本项目定义了两个自定义命令，简化知识入库流程：

### 1. /dehydrate - 原始文本脱水

用于处理已经复制好的原始文本（HTML 片段、富文本、乱码等）。

**使用方式：**
```
/dehydrate
[粘贴你的原始文本]
```

**功能：**
- 自动清洗 HTML 标签、属性和噪音
- 保留所有关键数据（数字、统计、年份、金额）
- 提取核心观点和关键证据
- 生成结构化 Markdown 文件保存到 `inbox/` 目录

**输出模板：**
```markdown
---
title: "{标题}"
date: {日期}
type: clipping
tags: [自动生成的标签]
---

# {标题}

> **一句话总结**：{摘要}

## 核心观点 (Key Takeaways)
- 观点1
- 观点2

## 关键数据与证据 (Fact Sheet)
- 数据1
- 数据2

---

## 原始文本清洗版

{清洗后的纯文本}
```

### 2. /save - 网页抓取与保存

用于直接保存网页内容，会自动抓取网页并生成结构化笔记。

**使用方式：**
```
/save [网页URL]
```

**功能：**
- 自动抓取网页内容并转换为 Markdown
- 智能提取标题、分类、摘要和标签
- 生成规范化文件名的 Markdown 文件

**输出模板：**
```markdown
---
title: "{标题}"
date: {日期}
original_url: "{原始URL}"
category: {分类}
tags: [{标签}]
---

# {标题}

> **摘要**：{摘要}

---

## 正文内容

{抓取的正文}
```

## 目录结构

```
.
├── inbox/                  # 待处理的内容（自动生成）
├── README.md              # 本文件
└── .opencode/
    └── commands/          # 自定义命令
        ├── dehydrate.md   # 脱水命令
        └── save.md        # 抓取命令
```

## 使用示例

### 示例 1：保存知乎文章

假设你看到一篇关于新疆农业技术的文章，直接复制全文后：

```
/dehydrate

[粘贴文章全文]
```

系统会自动生成 `inbox/2026-02-26-clip-xinjiang-agriculture.md` 文件。

### 示例 2：保存技术博客

看到一篇有用的技术文章：

```
/save https://example.com/article-about-docker
```

系统会自动抓取网页并生成结构化笔记。
